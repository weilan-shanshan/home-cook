# 私厨 — 家庭烹饪管理 PWA 技术方案

> 最后更新：2026-04-09
>
> 部署与排障请优先看：`private-chef/DEPLOYMENT_RUNBOOK.md`

## 一、项目概述

为家庭（5-10人）打造的私人菜谱管理与点餐应用。核心用户是做菜的人（记录菜谱、查看点餐）和吃饭的人（浏览菜单、点餐、评分）。

### 目标

- 快速上线，AI 全程开发
- 只给自家人用，零运维负担
- 苹果风 UI，简洁有品质
- 年费 ≤ 130 元

---

## 二、需求范围

### 一期功能（MVP）

| # | 功能 | 描述 |
|---|---|---|
| 1 | **菜品管理** | 多图上传、制作步骤、耗时、标签分类、记录每道做过的菜 |
| 2 | **家庭点餐** | 浏览菜单、标签筛选、勾选下单、生成点餐单 |
| 3 | **许愿菜单** | 家人许愿想吃什么（纯列表，不涉及 AI） |
| 4 | **烹饪日志** | 每次做菜记录时间、谁做的、评分评论、口味反馈 |
| 5 | **收藏夹** | 一键收藏喜欢的菜 |
| 6 | **通知** | 有人点餐 / 新菜品时，企业微信群机器人推送 |

### 二期功能（后续）

- 家庭广场（社交，其他家庭浏览菜单）
- AI 许愿（AI 推荐 / 生成菜谱）
- 分享卡片（生成精美图片发朋友圈）

### 不做

- 离线功能（家里 WiFi 稳定，不需要）
- 二期功能不在一期开发范围内

## 三、架构设计

### 整体架构

```
家人手机浏览器 (PWA 添加到桌面)
         │
         ▼
 Cloudflare Pages (前端静态资源, 免费 CDN, 无需备案)
         │
         │ API 请求 (HTTPS via Cloudflare Tunnel)
         ▼
 腾讯云轻量服务器 2C2G3M Ubuntu 22.04
 ┌──────────────────────────────────────┐
 │  Hono (Node.js)                      │
 │  ├── REST API                        │
 │  ├── Lucia v3 (Session 认证)         │
 │  ├── Drizzle ORM                     │
 │  └── SQLite (better-sqlite3, WAL)    │
 └──────────────────────────────────────┘
         │                    │
    ┌────┴────┐          ┌───┴───┐
    ▼         ▼          ▼       ▼
 腾讯云COS  企微Webhook  cron备份
 (图片存储) (通知推送)   (SQLite→COS)
```

### 图片上传流程

```
手机拍照
  → 前端 browser-image-compression 压缩至 ≤1MB
  → 前端请求后端 GET /api/upload/presign
  → 后端生成腾讯云 COS 预签名 URL 返回
  → 前端 PUT 直传 COS（不经过后端，不占 3Mbps 带宽）
  → 前端将 COS URL 提交给后端保存到 SQLite
```

### 认证流程

```
首次使用：
  厨师创建家庭 → 获得 6 位邀请码
  家人输入邀请码 + 填写昵称 → 注册成功 → 自动登录

登录态：
  Session Cookie (HttpOnly; Secure; SameSite=Strict)
  由 Lucia v3 管理，Session 存 SQLite
  Cloudflare Tunnel 提供 HTTPS，确保 Cookie 安全传输
```

## 五、数据模型

```sql
-- 用户表
CREATE TABLE users (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  username      TEXT NOT NULL UNIQUE,        -- 登录用名
  display_name  TEXT NOT NULL,               -- 显示昵称
  password_hash TEXT NOT NULL,               -- 密码哈希
  role          TEXT NOT NULL DEFAULT 'member', -- admin/member
  created_at    TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 家庭表
CREATE TABLE families (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  name        TEXT NOT NULL,                 -- 家庭名称
  invite_code TEXT NOT NULL UNIQUE,          -- 6位邀请码
  created_by  INTEGER NOT NULL REFERENCES users(id),
  created_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 家庭成员关联
CREATE TABLE family_members (
  family_id INTEGER NOT NULL REFERENCES families(id),
  user_id   INTEGER NOT NULL REFERENCES users(id),
  joined_at TEXT NOT NULL DEFAULT (datetime('now')),
  PRIMARY KEY (family_id, user_id)
);

-- 菜谱表
CREATE TABLE recipes (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  family_id   INTEGER NOT NULL REFERENCES families(id),
  title       TEXT NOT NULL,                 -- 菜名
  description TEXT,                          -- 简介
  steps       TEXT,                          -- 制作步骤 (JSON 数组)
  cook_minutes INTEGER,                     -- 预计耗时(分钟)
  servings    INTEGER DEFAULT 2,             -- 份量
  created_by  INTEGER NOT NULL REFERENCES users(id),
  created_at  TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 菜谱图片
CREATE TABLE recipe_images (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  recipe_id INTEGER NOT NULL REFERENCES recipes(id) ON DELETE CASCADE,
  url       TEXT NOT NULL,                   -- COS URL
  thumb_url TEXT,                            -- 缩略图 URL
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 标签表
CREATE TABLE tags (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  family_id INTEGER NOT NULL REFERENCES families(id),
  name      TEXT NOT NULL,                   -- 标签名：家常菜、川菜、汤类...
  UNIQUE(family_id, name)
);

-- 菜谱-标签关联
CREATE TABLE recipe_tags (
  recipe_id INTEGER NOT NULL REFERENCES recipes(id) ON DELETE CASCADE,
  tag_id    INTEGER NOT NULL REFERENCES tags(id) ON DELETE CASCADE,
  PRIMARY KEY (recipe_id, tag_id)
);

-- 烹饪日志（每次做菜记录一条）
CREATE TABLE cook_logs (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  recipe_id INTEGER NOT NULL REFERENCES recipes(id),
  cooked_by INTEGER NOT NULL REFERENCES users(id), -- 谁做的
  cooked_at TEXT NOT NULL DEFAULT (datetime('now')), -- 什么时候做的
  note      TEXT                              -- 本次做菜备注
);

-- 评分评论（关联烹饪日志）
CREATE TABLE ratings (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  cook_log_id INTEGER NOT NULL REFERENCES cook_logs(id) ON DELETE CASCADE,
  user_id     INTEGER NOT NULL REFERENCES users(id), -- 谁评的
  score       INTEGER NOT NULL CHECK(score >= 1 AND score <= 5), -- 1-5分
  comment     TEXT,                           -- 评论内容
  created_at  TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(cook_log_id, user_id)               -- 每人每次只能评一次
);

-- 许愿菜单
CREATE TABLE wishes (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  family_id  INTEGER NOT NULL REFERENCES families(id),
  user_id    INTEGER NOT NULL REFERENCES users(id), -- 谁许的愿
  dish_name  TEXT NOT NULL,                   -- 想吃什么
  note       TEXT,                            -- 备注
  status     TEXT NOT NULL DEFAULT 'pending', -- pending/fulfilled/cancelled
  recipe_id  INTEGER REFERENCES recipes(id), -- 实现后关联菜谱
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 点餐单
CREATE TABLE orders (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  family_id  INTEGER NOT NULL REFERENCES families(id),
  user_id    INTEGER NOT NULL REFERENCES users(id), -- 谁点的
  meal_type  TEXT NOT NULL,                   -- breakfast/lunch/dinner/snack
  meal_date  TEXT NOT NULL,                   -- 哪天的餐
  note       TEXT,                            -- 备注
  status     TEXT NOT NULL DEFAULT 'pending', -- pending/confirmed/completed
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 点餐单明细
CREATE TABLE order_items (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  order_id  INTEGER NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
  recipe_id INTEGER NOT NULL REFERENCES recipes(id),
  quantity  INTEGER NOT NULL DEFAULT 1
);

-- 收藏
CREATE TABLE favorites (
  user_id   INTEGER NOT NULL REFERENCES users(id),
  recipe_id INTEGER NOT NULL REFERENCES recipes(id) ON DELETE CASCADE,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  PRIMARY KEY (user_id, recipe_id)
);

-- Session 表 (Lucia v3)
CREATE TABLE sessions (
  id         TEXT PRIMARY KEY,
  user_id    INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  expires_at INTEGER NOT NULL
);
```

### SQLite 初始化配置（必须）

```typescript
const sqlite = new Database('./data/private-chef.db');
sqlite.pragma('journal_mode = WAL');        // 并发读写
sqlite.pragma('busy_timeout = 5000');       // 防 SQLITE_BUSY
sqlite.pragma('synchronous = NORMAL');      // 性能+安全平衡
sqlite.pragma('foreign_keys = ON');         // 外键约束
sqlite.pragma('cache_size = -8000');        // 8MB 缓存
```

## 九、部署方案

> **当前生效路径说明**
>
> - 当前前端：**Cloudflare Pages**
> - 当前后端：沿用现有服务器方案
> - 腾讯云一体化前后端同服：**当前不具备，不能视为现行方案**
>
> 本章节以下内容描述的是当前仍可执行的部署方式；如未来要切换到腾讯云一体化部署，应参考单独候选方案文档，而不是把这里理解为“即将切换”。

### 9.1 服务器初始化

```bash
# 1. 安装 Node.js (via nvm)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash
nvm install 22
nvm use 22

# 2. 安装 cloudflared (Cloudflare Tunnel)
curl -L https://pkg.cloudflare.com/cloudflared-linux-amd64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb

# 3. 配置 Cloudflare Tunnel
cloudflared tunnel login
cloudflared tunnel create private-chef
cloudflared tunnel route dns private-chef api.weilanshanshan.top

# 4. 安装 pm2 进程管理
npm install -g pm2
```

### 9.2 后端部署

```bash
# 构建
cd backend && npm run build

# pm2 启动
pm2 start ecosystem.config.cjs --env production

# cloudflared 启动（指向本地端口）
cloudflared tunnel run --url http://localhost:3000 private-chef
```

### 9.3 前端部署

```bash
# 构建
```

### 10.3 实现要点

```typescript
// lib/wechat.ts
const WEBHOOK_URL = process.env.WECHAT_WEBHOOK_URL;

// 消息队列：防止超过 20条/分钟 限流
const queue: string[] = [];
let sending = false;

async function sendToWeChat(content: string) {
  queue.push(content);
  if (sending) return;
  sending = true;

  while (queue.length > 0) {
    const msg = queue.shift()!;
    const res = await fetch(WEBHOOK_URL, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        msgtype: 'text',
        text: { content: msg }
      })
    });

    const data = await res.json();
    // 必须检查 body 而非 HTTP status（企微总是返回 200）
    if (data.errcode === 45009) {
      // 限流，等 60 秒重试
      queue.unshift(msg);
      await new Promise(r => setTimeout(r, 60000));
    } else if (data.errcode !== 0) {
      console.error('WeChat bot error:', data);
    }

    // 滑动窗口：间隔 3 秒发送
    await new Promise(r => setTimeout(r, 3000));
  }

  sending = false;
}
```

## 十一、关键风险与对策

| # | 风险 | 严重度 | 对策 |
|---|---|---|---|
| 1 | 客户端图片未压缩导致上传超时 | 🔴 高 | 强制 browser-image-compression 压缩至 ≤1MB |
| 2 | 裸 IP 后端导致 HTTPS/Cookie 失败 | 🔴 高 | 使用 Cloudflare Tunnel 提供 HTTPS |
| 3 | 3Mbps 带宽图片服务瓶颈 | 🔴 高 | 图片走 COS 预签名直传，不经过服务器 |
| 4 | SQLite 并发写入 BUSY 错误 | 🟠 中 | WAL 模式 + busy_timeout=5000 |
| 5 | 企微 bot 限流（20条/分钟） | 🟠 中 | 消息队列 + 滑动窗口限流 |
| 6 | 单 VPS 数据丢失 | 🟠 中 | cron 每日 SQLite 备份到 COS |
| 7 | QQ 浏览器不支持 backdrop-filter | 🟡 低 | @supports 检测 + 纯色降级 |
| 8 | iOS Safari PWA 7天缓存清除 | 🟡 低 | 不需要离线功能，影响极小 |

## 十二、AI 代码生成注意事项

以下是 AI 生成代码时常见的错误模式，开发时需特别检查：

### 必须做

- 所有 SQL 查询使用参数化（better-sqlite3 天然支持）
- 所有密钥/Webhook URL 放 `.env`，不硬编码
- 图片上传使用 `XMLHttpRequest`（fetch 无上传进度 API）
- Service Worker 对 `/api/*` 使用 `NetworkFirst` 策略
- 认证中间件覆盖所有非公开路由（包括图片路由）
- SQLite 使用 Drizzle 迁移文件管理 schema 演进
- 毛玻璃 CSS 使用 `@supports (backdrop-filter: blur(1px))` 包裹
- 企微 bot 检查响应 body 的 `errcode`，不看 HTTP status

### 不能做

- 不使用 `as any`、`@ts-ignore`、`@ts-expect-error`
- 不使用 `CREATE TABLE IF NOT EXISTS` 内联建表
- 不使用 Base64 编码上传图片（体积膨胀 33%）
- 不在 `localStorage` 存认证 token
- 不对 sticky 全宽元素加 `backdrop-filter: blur()`（严重卡顿）
- 不创建 `URL.createObjectURL()` 后忘记 `revokeObjectURL()`
- 不在循环中查询数据库（N+1 问题）
