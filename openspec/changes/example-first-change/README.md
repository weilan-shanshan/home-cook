# Example First Change

This is a starter change template you can copy for future work.

Recommended workflow:

1. Rename the folder to a real kebab-case change name.
2. Fill in `proposal.md` with user problem, scope, and success criteria.
3. Update only the delta spec files for modules affected by the change.
4. Add `design.md` details only when the change is cross-module or non-trivial.
5. Use `tasks.md` as the implementation and verification checklist.
