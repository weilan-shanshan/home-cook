## 3. Critical Gotchas & Learned Patterns

### Architecture
- **Cookie Auth**: The app uses `HttpOnly; SameSite=Strict` session cookies. `credentials: 'include'` is required for all frontend requests. Cookies are marked `Secure` in production.
- **Hono AppType Pitfall**: Exporting `AppType` from Hono can fail to infer routes if not chained correctly. The frontend uses a custom `apiFetch` wrapper (`frontend/src/lib/api.ts`) to maintain type safety without depending on brittle Hono RPC exports.
- **No Bare Fetch**: Direct `fetch()` calls are banned by AST rules. Always use `apiFetch` in hooks.

### Database
- **SQLite Concurrency**: WAL mode is enabled in `backend/src/db/index.ts`.
- **Foreign Key Constraints**: Explicitly enabled via `PRAGMA foreign_keys = ON`.
- **Delete Cascade**: `recipes` deletion cascades to `recipe_tags` and `recipe_images`, but `cook_logs` and `orders` require manual handling to avoid constraint violations.

### Storage (COS)
- **CORS**: COS bucket `weilan-1254036222` must have CORS rules allowing `http://localhost:5173` and `https://weilanshanshan.top`.
- **Presigned PUT**: Frontend uploads directly to COS using presigned URLs generated by the backend to save server bandwidth.

## 4. Deployment Reality & Blockers

### Current Environment
- **API Base**: Production API base URL is resolved via `VITE_API_BASE_URL` in `.env.production` (used by `frontend/src/lib/api.ts`).
- **DNS**: Domains `weilanshanshan.top` and `api.weilanshanshan.top` currently have no DNS resolution in this environment.
- **SSH/Cloudflare**: No credentials or tunnel access available.

### Infrastructure Mapping
- **Frontend**: Target is Cloudflare Pages (dist folder).
- **Backend**: Target is Ubuntu 22.04 server managed via PM2.
- **Database**: Local SQLite file at `backend/data/private-chef.db`.
