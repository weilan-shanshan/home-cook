# Skills 索引模板

> **用途**: 只保存可复用工作流的轻量索引，不保存全文。

| Skill 名称 | 适用问题 | 触发信号 | 加载后预期收益 | 不该何时用 |
|---|---|---|---|---|
| 示例：ajx-builder | AJX 构建、启动 dev server | “构建 AJX 项目” | 快速进入可调试状态 | 普通前端项目 |
| 示例：ajx-debugger | AJX 真机调试 / 抓包 | “AJX 白屏 / 网络请求” | 直接进入设备与 CDP 调试 | 纯 Node 服务 |

## 维护规则

- 只记录 **我学会了怎么做这类事** 的流程。
- 只放索引，不放超长正文。
- 当 Skill 已过期或被替换时，更新索引而非静默保留。

## 与其他层的边界

- 不属于 `MEMORY.md`：Skill 是方法，不是事实。
- 不属于 `session_search`：Skill 是复用套路，不是情景回忆。
- 不属于 `USER.md`：Skill 是 Agent 能力，不是用户画像。

---
*Skill 的目标是按需加载，而不是常驻挤占提示词。*
