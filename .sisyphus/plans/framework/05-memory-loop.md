# 05-记忆循环：四层记忆与冷热分层 (Memory Loop)

> **核心逻辑**: 不是所有记忆都应该进入提示词。越热、越稳定的信息越前置；越冷、越偶发的信息越按需检索。

## 1. 四层记忆模型 (Four Layers)

### Layer 1 — 提示词热记忆

- **`MEMORY.md`**: Agent 视角的长期稳定事实。
- **`USER.md`**: 用户画像、长期偏好、稳定协作方式。
- **特点**:
  - 每轮对话启动时进入固定提示词前缀。
  - 必须体量小、变化慢、命中高。
  - 推荐使用字符数上限管理，而非 token 预算。

### Layer 2 — 会话检索 (`session_search`)

- 解决“我们之前聊过什么”的问题。
- 适合情景回忆，不适合持久事实。
- 应优先用于历史对话、上次方案、过往结论的回查。

### Layer 3 — Skills 程序性记忆

- 解决“这类事情通常怎么做”的问题。
- 本质是可复用工作流，而非某次任务的结果。
- 固定前缀只保留 Skills 索引，完整内容在命中后再加载。

### Layer 4 — Honcho（可选）

- 解决跨会话、更深层的用户画像问题。
- 只有在用户长期画像真的能提升任务质量时才启用。
- 不应默认把所有人物信息都放进这一层。

## 2. 热记忆内容边界

- **`MEMORY.md` 应存**:
  - 项目环境信息
  - 长期稳定工程约束
  - 高频复用的项目事实
- **`MEMORY.md` 不应存**:
  - 任务进度
  - 临时排障过程
  - 本轮会话中间状态
- **`USER.md` 应存**:
  - 长期语言偏好
  - 稳定沟通风格
  - 长期可预期的工作习惯
- **`USER.md` 不应存**:
  - 当前任务的临时要求
  - 一次性情绪或短期偏好

## 3. 持久化知识载体（项目侧）

> 这一层不是 Hermes 的“热记忆层”，而是**项目知识仓库**。它服务于工程沉淀与检索，不直接等同于固定提示词前缀。

框架仍保留项目侧的 4 个标准知识文件，存放在 `.sisyphus/notepads/` 下，用于沉淀工程知识：

- **`decisions.md`**: 
  - 记录所有的 **架构/逻辑决策**。
  - 记录所有的 **契约变动**。
  - 记录所有的 **权限/安全策略**。
- **`learnings.md`**: 
  - 记录所有的 **技术坑点** 和对应的 **解决方案**。
  - 记录所有的 **隐性业务规则**（如：第一个注册的用户是 Admin）。
  - 记录所有的 **第三方库版本陷阱**。
- **`issues.md`**: 
  - 记录所有的 **未决问题**。
  - 记录所有的 **已知 Bug**（暂时无法修复或范围外）。
  - 记录所有的 **待讨论事项**。
- **`implementation-knowledge-base.md`**: 
  - 记录项目特定的 **实现逻辑** 和 **代码规范**。
  - 记录 **常用工具类** 的用法。
  - 记录 **核心数据模型** 的业务含义。

## 4. 写入与更新语义

对热记忆，建议采用 3 种操作语义：

- **add**: 新增一条此前不存在、且值得常驻的稳定事实。
- **replace**: 用新事实替换旧事实，保持前缀紧凑。
- **remove**: 删除过时、低命中或错误的条目。

> 不建议把热记忆当日志顺序追加；它更像一份精选集，而不是会话流水账。

## 5. 记忆同步工作流 (Sync Workflow)

1. **触发 (Trigger)**:
   - 任何非显式指定的逻辑选择（决策）。
   - 任何超过 15 分钟的调试过程（学习）。
   - 任何需要推迟处理的问题（Issue）。
   - 任何值得进入热记忆的长期稳定事实。
2. **记录 (Log)**:
   - 长期稳定事实 → 写入 `MEMORY.md` / `USER.md`。
   - 情景回忆需求 → 不写热记忆，改走 `session_search`。
   - 可复用工作流 → 更新 Skills 索引。
   - 项目知识沉淀 → 使用 [记忆条目模板](./templates/memory-entry-template.md) 写入对应的 Notepad。
3. **引用 (Reference)**:
   - Agent 在开始任务前，必须先判断需要的是“事实”“情景”还是“方法”。
   - 项目知识优先搜索 `notepads/`；历史对话优先 `session_search`；方法问题优先 Skills。
4. **清理 (Refinement)**:
   - 定期合并重复项。
   - 将已解决的 Issue 标记为 `Closed` 并归档。

## 6. 项目知识等级 (Project Knowledge Levels)

- **Level 1 (Ephemeral)**: 仅在当前会话中，适合 Todo List 的子步骤。
- **Level 2 (Task-specific)**: 记录在任务详单的 `Notepad` 部分，影响该任务。
- **Level 3 (Persistent Project Knowledge)**: 记录在 `.sisyphus/notepads/`，影响整个项目。

## 7. 防错边界 (Guardrails)

- **禁止过时记忆**: 任何被新决策覆盖的旧决策，必须明确标记为 `DEPRECATED`。
- **禁止冗长热记忆**: `MEMORY.md` / `USER.md` 只保留高命中率事实，不能膨胀成知识库。
- **强制同步**: 任何导致 `private-chef.md` 或 `README.md` 更新的动作，必须同步更新 `decisions.md`。
- **禁止混层**: 持久事实、情景回忆、可复用流程、深层画像必须分层维护。

---
*记忆不是越多越好，而是越分层越高效。*
